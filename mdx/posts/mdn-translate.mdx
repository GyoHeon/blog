## mdn

mdn은 엄청나게 많은 문서를 보유하고 있는 웹 개발자들의 백과사전이다.
mdn의 수 많은 문서들은 보통 여러 언어로 된 번역본을 제공해준다.
한국에서도 mdn 번역 팀을 주축으로 mdn 문서 번역이 이루어지지만, 문서의 수가 너무나도 많기 때문에 번역이 되지 않았거나 구버전의 번역본이 존재하는 경우가 많다.
mdn 문서 번역을 통해 자주 사용하는 오픈 소스에 기여해보자!

### 개발 환경 세팅

(`git`, `Node.js`, `Yarn`이 설치되어 있어야한다.)

가장 먼저 [**mdn/content**](https://github.com/mdn/content)와 [**mdn/translated-content**](https://github.com/mdn/translated-content)를 fork한다.
번역은 **mdn/transtlated-content**에서 진행한다.
**mdn/content**는 원문을 확인하고, dev 모드에서 미리 번역 결과물을 확인하는 데 사용한다.

**mdn/content**의 dev 모드를 사용하기 위해선 간단한 작업이 필요하다.

1. **mdn/content**의 root 디렉토리에서 `.env` 파일을 생성한다.
2. `.env` 파일에 `CONTENT_TRANSLATED_ROOT={translated-content의 디렉토리 위치}`를 추가한다.
   - VSCode를 사용할 경우 `EDITOR=code`도 추가해준다.

그후 **mdn/content**, **mdn/translated-conten**에서 `yarn install`을 실행한다.

### 번역하기

1. 아직 번역되지 않았거나 구버전으로 번역되어 있는 문서를 찾는다.
2. **mdn/translated-content**에서 branch를 만든다.
3. 번역한다.
   - 번역의 스타일을 가이드해주는 문서가 아주 많다. 아래 참조에 모아놨다.
4. **mdn/content**에서 yarn start를 하면 번역 결과를 실시간으로 확인할 수 있다.

### 미리 linting 하기

번역을 한 문서가 markdown lint에 어긋나면 당연히 merge할 수 없다.
그러므로 미리 lint를 하는 것이 불필요한 소통을 줄일 수 있다.
해당 과정은 PR을 리뷰해주신 [**@wisedog**](https://github.com/wisedog)님이 추천해주셨다.

1. `markdownlint-cli2`를 설치한다.
2. terminal에서 `yarn lint`를 실행한다.

```bash
markdownlint-cli2 "{수정한 파일 위치 + 파일 이름}"
```

### PR 보내기

1. 번역이 완료되면 **mdn/translated-content**로 PR을 보낸다.
2. PR은 mdn 한국 번역 팀 분들의 리뷰가 필요하다.
   - 적절한 리뷰를 해주시거나 merge를 해주신다.

### 끝!

Merge가 되면 끝!
간단하면서도 뿌듯한 오픈 소스 기여다!

너무 간단한 내용이라서 부끄럽지만 필자의 PR을 첨부한다.

interface와 type은 타입스크립트에서 특정한 타입을 지정할 때 사용한다.
이전에는 분명히 객체를 다룰 땐 interface가 장점이 있다는 생각을 했는데, 코드 리뷰를 진행할 때 명확한 답을 못 해서 서칭해보았다.
그런데 이제는 아니란다!!

## ??

근본적으로 interface는 객체의 타입을 지정하는 용도이니, type과 비교하면 당연히 기능의 다양성이 떨어진다.
근데 왜 interface가 있는가 하니...

## interface가 형임

interface가 type보다 먼저 나와서 그렇다.
타입스크립트가 진화함에 따라 단순한 타입, interface를 넘어서는 기능이 필요해졌고, type이 등장했다.
물론 초반에는 type이 interface보다 느리고 했다는데 지금은 그렇지도 않다고 한다.

## 그래도 쓰기 편하니까

하지만 아직도 interface가 엄청난 우위를 보이는 종목이 있는데 그것은 바로 가독성이다.
interface로 User라는 타입을 이용해 GameMaster라는 타입으로 확장시키는 방법은 아래와 같다.

```ts
interface User {
  name: string;
  age: number;
}

interface GameMaster extends User {
  master: boolean;
}
```

똑같은 동작을 type으로 하려면 아래와 같다.

```ts
type User = {
  name: string;
  age: number;
};

type GameMaster = User & {
  master: boolean;
};
```

근데 위 두개는 약간 다르다.
타입은 순서가 중요하다.
